### MEDIUM — Dependency Vulnerability: `rustls-webpki 0.103.9`

Flagged independently by **Trivy** and **OSV-Scanner**.

| Advisory | CVSS | Description | Fixed Version |
|----------|------|-------------|---------------|
| RUSTSEC-2026-0049 / GHSA-pwjx-qhcg-rvj4 | 4.4 | CRLs not considered authoritative by Distribution Point due to faulty matching | 0.103.10 |
| RUSTSEC-2026-0098 | — | Follow-up CRL fix | 0.103.12 |
| RUSTSEC-2026-0099 | — | Follow-up CRL fix | 0.103.12 |

**Impact:** Medium. `rustls-webpki` handles TLS certificate validation. rtk is a CLI proxy that does not make direct TLS connections, so practical exploitability is low. Fixed upstream minimum version is 0.103.12 (resolves all three advisories).

## Fix Applied

**File:** `Cargo.toml`

Added exact version pin under `[dependencies]`:

```toml
[dependencies.rustls-webpki]
version = "=0.103.12"
```

This forces Cargo to resolve `rustls-webpki` to 0.103.12 throughout the full dependency tree, replacing the transitive 0.103.9 pulled in by `ureq`.
